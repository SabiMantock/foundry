---
description: Promote a proven pattern from product code into a reusable Registry module.
argument-hint: <pattern / file / feature to harvest>
---

Run the **harvest-module** skill on: **$ARGUMENTS**

Launch the `harvester`:
1. Confirm it's genuinely net-new (reuse-search) and has a real or imminent 2nd consumer.
2. Generalize the pattern — strip first-instance quirks; push market-specifics into an adapter.
3. Package to the `example-hello` template: contract, internals, unit + contract tests, README,
   `foundry` metadata at `tier: experimental`.
4. Add the catalog entry and hand to `registry-librarian`.

Then present it to the operator for admission. Promotion above `experimental` requires ≥2 real
consumers and your sign-off later.
